## 安装故障

### 现象：安装 TMA 时防病毒软件提示风险

**故障原因：**TMA 安装时会修改系统配置和注册表，防病毒软件提示风险属正常现象。

**解决办法：**在防病毒软件提示风险时选择"允许操作"。

### 现象：客户端安装失败

**故障原因：**在不支持的操作系统版本上安装 TMA。

**解决办法：**使用支持的操作系统重新安装。TMA 支持 64 位操作系统：

- Windows：Windows 7、Windows 10、Windows 11。
- Mac：macOS 10.10 (Yosemite) 及以上。
- iOS：iOS 15.0 及以上。
- Android：Android 6.0 及以上。

> 注意：Windows 7 SP1 需安装补丁 KB4474419。TMA 标准版不支持 32 位操作系统和 Linux，如有需求请联系定制化版本。

### 现象：安装时导入证书失败

**故障原因：**TMA 导入证书需与 TM 企业注册平台通信，导入失败可能是互联网访问异常。

**解决办法：**检查本地网络，确认互联网连接正常后重新安装。

## 认证故障

### 现象：使用企微等第三方扫码登录，二维码加载慢或加载失败

**故障原因：**TMC 调用第三方接口获取二维码，本地网络质量不佳导致。

**解决办法：**检查本地网络，更换网络质量更好的网络。

## 网络故障

### 现象：终端环境检测异常

**故障原因：**不符合终端入网规则配置（系统版本/软件/进程/补丁/defender/域控等限制）。

**解决办法：**按照提示的不合规内容进行终端环境优化。

### 现象：同步安全云网失败

**故障原因：**

- 原因 1：本地网络问题，获取云网配置失败。
- 原因 2：无可用 TM 网关。
- 原因 3：云网虚拟 IP 与本机 IP 冲突。
- 原因 4：创建虚拟网卡失败。

**解决办法：**

- 办法 1：检查本地网络质量。
- 办法 2：在终端执行网络监测，检查与 TMG 连通性，超时请联系管理员。
- 办法 3：重启客户端后重新登录入网，TMA 将重新分配虚拟 IP。
- 办法 4：检查本机是否安装流量抓包工具（如 Wireshark），升级至最新版后重新登录 TMA。

### 现象：加入安全云网失败

**故障原因：**

- 原因 1：本地网络问题导致 TMA 与 TMG 连接探测多次失败。
- 原因 2：本地启用了其他网络代理程序（如 VPN），通过默认路由转发了网络流量。

**解决办法：**

- 办法 1：选择质量更好的本地网络，使用网络检查功能检查本地网络质量，始终与 TMG 延时较大或超时请联系管理员。
- 办法 2：检查并关闭本地其他网络代理软件，重启客户端重新登录入网。

## 业务访问故障

### 现象：获取不到资源列表

**故障原因：**本地网络质量不佳。

**解决办法：**

1. 点击资源组旁边的刷新按钮重新刷新业务资源。
2. 点击在线时长旁边的重置网络按钮重置网络配置。
3. 选择质量更好的本地网络，退出 TMA 并重新登录。

### 现象：资源访问缓慢或失败

**故障原因：**本地网络质量不佳，或终端到 TMG 的网络质量不佳。

**解决办法：**检查或切换本地网络，或退出 TMA 并重新登录。

### 现象：Web 业务资源可以 ping 通，但浏览器无法打开

**故障原因：**浏览器使用了代理插件。

**解决办法：**检查并关闭浏览器 Web 代理插件后重新访问。

### 现象：资源页面打开时提示"您的连接不是私密链接"

**故障原因：**业务网站证书出现错误或未使用正确证书。

**解决办法：**使用正确的 SSL 证书代理 HTTPS 业务资源。无法提供证书时：

- Chrome 浏览器：在页面键入"thisisunsafe"。
- Safari 浏览器：点击"显示详细信息"，再点击"访问此网站"。
- IE 或 Edge 浏览器：点击"详细信息"，再点击"转到此网页（不推荐）"。
- Firefox 浏览器：点击"高级"，再点击"接受风险并继续"。
