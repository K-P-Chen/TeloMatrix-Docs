## 概述

TeloMatrix云网（简称 TM）在物理网络之上构建一张点对点虚拟网络，将企业分散在各地的终端和业务资源整合成一个逻辑网络。核心机制：网络访问控制基于身份而非 IP 地址。

---

## 解决的问题

1. **远程访问体验差**：VPN 每次只能连接一个网关，跨区域访问需手动切换，流量绕行总部导致延迟高、带宽浪费。
2. **互联网暴露面大**：VPN 网关和业务端口直接暴露在公网，易被扫描和攻击。
3. **安全策略难维护**：基于 IP 做访问控制，IP 仅代表位置不代表身份，分布式环境下策略配置越来越复杂，隐含信任漏洞增多。

TM 的解决方式：虚拟网络中终端与业务资源默认隔离，只有通过身份认证且获得明确授权后才能互访。网络对外部完全隐身，未授权者无法感知该网络存在。

## 四个核心机制

### 1. 虚拟化组网

在所有终端和业务资源之间建立 Full-Mesh 结构的 Overlay 网络。每个终端与所有业务资源逻辑上点对点直连，不经过中间节点。终端可同时访问分布在任意位置的多个业务资源，无需切换链路。

<img src="/images/1_About_TM/1_About_TM/TM_Design_Philosophy/2.Network_Virtualization.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

### 2. 数据包实名制

将企业统一身份源（AD/LDAP、钉钉、企微、飞书等）同步到云网。终端实名注册后，身份信息植入每个传输层数据包。网络上传输的每个数据包均可追溯到具体的人和终端，访问控制不再依赖 IP 地址。

<img src="/images/1_About_TM/1_About_TM/TM_Design_Philosophy/3.Unified_Identity_Real_Name_Data_Network.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

### 3. 集中编排 + 分布式执行

管理员在控制台用业务语言配置策略（哪个帐号、什么终端、访问哪个业务资源）。TM 根据当前环境动态生成细粒度访问控制策略，同步到各终端和网关分布式执行。无集中式网关性能瓶颈。

<img src="/images/1_About_TM/1_About_TM/TM_Design_Philosophy/4.Centralized_Policy_Orchestration_Distributed_Execution.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

### 4. 身份化行为分析

所有终端环境数据和网络流量日志均打上身份标签。TMI 分析平台用 AI 对每个帐号和终端的访问行为建立基线（维度包括访问时间、地点、协议类型、频率、敏感数据访问等），行为偏离基线时自动研判，确认威胁后全网处置。

## 组件一览

TM 由 6 个组件构成，控制平面与数据平面分离：

| 组件 | 作用 | 部署要点 |
|------|------|----------|
| **TMC** 管理控制平台 | 帐号/终端/业务资源管理、策略编排、认证授权 | 任意位置，不对外开端口 |
| **TMG** 网关 | 代理业务资源接入，执行访问控制和流量采集 | 数据中心/云环境，仅开一个 UDP 端口，支持集群 |
| **TMA** 客户端 | 终端轻量级代理，身份认证、环境检查、访问转发 | Windows/macOS/iOS/Android |
| **TMD** 域名解析 | 云网内部 DNS 服务，不修改企业原有 DNS | 任意位置，不对外开端口 |
| **TMI** 智能分析 | 身份化数据分析、行为基线、威胁研判 | 任意位置，不对外开端口 |
| **TME** 边缘网关 | 无端模式：浏览器访问 Web/SSH/RDP 资源 | 互联网边缘，无需安装客户端 |

## 与传统方案对比

### 与 VPN

**相同**：均支持远程访问内网资源，均对公网流量加密传输。

**不同**：

- **网络结构**：VPN 为星型（终端连网关，同时只连一个），TM 为 Full-Mesh（终端同时与所有网关直连）。VPN 多区域访问需绕行或切换，TM 不需要。
- **安全边界**：VPN 基于 IP 和物理网络边界，连上后可访问网段内所有资源。TM 基于身份，连上后默认无任何访问权限，逐条授权后资源才可见。
- **持续验证**：VPN 仅在接入时验证一次，TM 持续监控终端环境和行为，异常时触发二次验证或自动收权。

### 与零信任架构

零信任核心理念（身份为访问控制基础、最小权限、持续评估）TM 均有对应实现。TM 侧重底层网络改造——将身份植入数据包，在传输层执行实名的分布式访问控制。

<img src="/images/1_About_TM/1_About_TM/TM_and_Zero_Trust/2.TM_Zero_Trust_Network_Access_Process.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

### 与 SDP

SDP 是零信任的一种实现（三组件：Client、Gateway、Controller）。TM 与 SDP 结构上有相似之处：控制平面/数据平面分离、先认证后访问的模式。

关键差异：

- **网络层**：SDP 为星型结构，依赖 mTLS + 单包授权（SPA）做端口隐藏和身份校验，Controller 集中校验存在性能瓶颈。TM 为去中心化 Mesh 网络，每个数据包在两端间逐包认证，Controller 不参与数据面验证。
- **访问控制**：SDP 基于 IP 做网络层控制；TM 基于身份做每包访问控制，粒度更细，天然分布式执行。

<img src="/images/1_About_TM/1_About_TM/TM_and_SDP/3.FullMesh.png" style={{width: "50%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

### 与 SASE

SASE 是 Gartner 2019 年提出的概念，主张网络能力与安全能力融合为云服务（SWG、CASB、ZTNA 等统一交付）。

TM 与 SASE 相似之处：身份驱动、全球分布。路线差异：

- SASE 将流量牵引到安全公有云检测，企业业务流量在云上以明文处理，存在隐私风险和额外带宽成本。
- TM 采用私有化方案，业务流量在终端与客户自有网关间点对点全程加密，中途不解密，数据始终由企业自治。

如果抛开"安全能力集中到云上"这个前提，TM 可视为一种客户完全私有的 SASE 实现。

<img src="/images/1_About_TM/1_About_TM/TM_and_SASE/1.SASE1.png" style={{width: "60%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

### 与 CSMA

CSMA（网络安全网格）是 Gartner 2021-2022 年提出的架构理念，强调分布式架构中集中编排、分布式执行，身份成为安全边界。

TM 架构与之契合：集中策略编排 + 分布式访问控制、身份化数据包、多源数据综合分析。

<img src="/images/1_About_TM/1_About_TM/TM_and_CSMA/1.2021CSMA.png" style={{width: "60%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

定位差异：CSMA 描述整合现有安全系统的上层架构；TM 定位于底层虚拟化网络基础设施——通过虚拟化整合碎片化网络，将身份植入数据包，为上层安全系统提供管控和数据接口。

## 一句话总结

TM：在现有物理网络之上，用虚拟化技术构建一张**点对点直连、数据包实名、对外完全隐身的私有安全网络**。员工在任何地点安全访问业务，管理员用身份而非 IP 管控网络。
