## 需求背景

随着企业数字化转型需求日益增多，“上云、用数、赋智”让更多分布式企业采用多云和混合云架构，云上业务的互联互通方式各异。

云网融合的趋势和需求带来了新的挑战，即分布式复杂网络如何统一运维和管理。


## 场景实例

### 现状描述

1. **碎片化企业网络**

  某大型互联网企业拥有近两万员工，设有国内总部、国内分公司和海外分公司。
  
  主要的内部业务资源部署在数据中心A和数据中心B，对外开放业务部署在公有云A和公有云B上。

  数据中心与公有云之间用专线打通，数据中心之间也用专线互联，确保业务带宽稳定性和安全性。

2. **总部员工接入**

  总部办公区分别与两个数据中心用专线互联，办公区员工通过专线访问数据中心的业务资源。
  
  总部办公区划分多个办公区安全域，正式员工根据接入方式和部门归属，接入到不同的安全域中。

  外包员工归属于单独的安全域，设置单独的安全策略。

3. **渠道合作伙伴接入**

  渠道合作伙伴办公区通过IPSec VPN接入数据中心访问业务资源。

4. **分公司办公区接入**

  国内分公司办公区员工通过专线访问数据中心A上的业务资源。

  海外分公司办公区通过SSL VPN接入到总部办公区，通过总部VPN跳转访问数据中心的业务资源。

5. **移动办公接入**

  移动办公人员通过SSL VPN接入到总部办公区的VPN专区，然后跳转访问数据中心的资源。

6. **统一身份认证和管理**

  总部办公区部署了AD域，对全公司正式员工的帐号进行统一认证和管理。部署了单点登录系统（SSO），所有员工在访问应用服务的时候都通过单点登陆系统进行授权和应用登录。

### 需求分析

1. **减少专线带宽成本**

  目前总部和国内分公司都是通过专线接入数据中心，并且所有员工通过VPN访问时都需要先接入到总部办公区的VPN网关，随后跳转访问数据中心业务资源。这样导致接入总部的专线带宽耗用高，成本高。

2. **提高终端用户体验**

  - 居家办公、移动办公、渠道合作伙伴远程接入都是通过SSL VPN先接入到总部办公区，然后从总部VPN接入区跳转到数据中心业务资源，会导致访问延时以及稳定性的单点失效风险。

  - 海外员工通过总部VPN接入总部后跳转访问数据中心业务，跨境链路延时导致页面响应时间长，用户体验差。

  - 内部业务系统多，员工记不住大量的业务系统帐号密码，需要用单点登录系统对业务系统的登录认证进行统一接管，提高用户体验。

  - 渠道合作伙伴仅需访问少量业务资源，但同样用VPN接入在网络层面就是接入整个内网，引入新的安全风险。

3. **提高安全管控效率**

  - 数据中心、公有云、总部和分支办公区域的异构组网使得物理网络结构复杂，外加内外用户数量多角色杂，业务资源数量多变化快，授权和访问控制策略管理起来时效性低且非常复杂，需要业务部门、安全部门和网络管理部门通力协作，管理成本高。因此，企业亟需能简化的全网访问策略管理方式，减少管理复杂度，降低出错率，提高行营效率。

  - 公司每个员工一般都有办公PC和手机两个终端，有时还涉及员工自己的PC远程访问，终端管控一直以来都是公司头疼的问题。因此需要有效方案对全网终端进行集中管控，对终端进行盘点，了解掌握每个办公终端的安全状况，对终端进行策略授权管控。

4. **对业务访问行为进行监控**

  公司员工多且分散，人员复杂且流动性大。公司需要对每一个员工的业务访问行为进行审计，尤其是核心敏感的业务系统的访问行为进行审计，实时监测异常访问或恶意访问行为，实时自动地进行告警和处置。从而避免内部数据泄露造成安全事件，且所有安全事件都能追溯到具体的人和终端。

### 部署方案

1. **TM 安全控制平台TMC部署**

  - 在总部服务器区部署两台TMC，互为热备，配置内网IP地址，禁止外部网络的访问。

  - 在TMC上配置AD身份源，定时同步AD组织架构和帐号，同时在TMC上为渠道合作伙伴配置独立身份。

  - 在TMC配置业务资源的单点登录接口，与公司已建SSO对接。

2. **TM 域名解析服务TMD部署**

  - 在总部服务器区部署TMD，配置内网IP地址，禁止外部网络的访问。

  - TMD上级DNS查询服务指向公司内网DNS。

3. **TM 智能分析平台TMI部署**

  在总部服务器区部署TMI，配置内网IP地址，禁止外部网络的访问。

4. **TM 网关TMG部署**

  - 在总部服务器区、公有云、数据中心分别部署两台TMG，每台TMG分别代理所在区域的所有业务资源。

  - 在公有云A上申请一个具备跨境加速能力的虚拟机，并在这个虚拟机上单独部署一台TMG。在总部服务器区的TMC上配置策略，让所有海外员工的终端在访问国内业务资源时，下一跳都指向到这台具备跨境加速能力的TMG。

5. **TM 边缘网关TME部署**

  在数据中心A部署一台TME，代理所有渠道合作伙伴需要访问的业务资源。

6. **TM 客户端TMA部署**

  所有公司员工办公终端上都安装TMA。

### 实现效果

1. **简化复杂的分布式网络，降低运维管理成本**

  - TM 云网将复杂的分布式网络简化成点到点全网互联的虚拟网络，管理员无需再考虑中间物理网络的复杂性，可基于AD同步的帐号、组织架构等进行业务访问授权。

  - 授权策略可基于组织架构、角色、终端类型等因素，管理员无需考虑企业人员的日常变动。当AD帐号出现变化，员工在全网的访问控制策略会自动进行计算和同步，减少策略维护工作量。

  - 管理员可以在TMC上配置业务管理员帐号，下放业务发布和访问策略编排权限，授权业务部门员工在设定范围内自行发布业务资源，配置授权策略，从而允许员工访问。在安全可控的前提下，提高了业务方的业务灵活性，减少了安全管理员和网络管理员的工作负荷。

2. **减少带宽成本**

  - 远程办公员工、海外分公司员工无需通过VPN接入公司总部作为跳转后再访问数据中心的业务，大量减少公司总部的出口带宽耗用，降低成本。

  - 国内分公司员工、渠道合作伙伴无需使用专线和VPN接入数据中心资源，减少了专线和VPN的带宽成本。

3. **互联网暴露面收敛，降低外部安全风险**

  - 公司所有内部业务资源都无需暴露于互联网，也无需对内网办公区暴露，而唯一需要做的时TMG网关对外开放一个UDP端口，且这个UDP端口只接收通过身份验证的数据包。最大可能地缩小了业务资源的对外暴露面，最大限度最小成本地规避了来自不可信网络地安全威胁。

4. **提升员工地业务访问体验**

<img src="/images/1_About_TM/Customer_Cases/Distributed_Complex_Network_Unified_Management/3.Implementation_Effect.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

  - 使用不同终端地用户可以用不同接入方式访问业务资源：

    - 海外员工使用TMA访问国内业务资源，业务访问流量先经过跨境加速节点，提高访问速度；

    - 渠道合作伙伴可以不安装TMA用通用第三方浏览器访问TME到统一门户，经过多因素认证后访问有限开放的业务资源；

    - 公司员工都使用TMA访问业务资源，无论是在总部、分公司、差旅途中、居家办公，只要打开TMA通过认证授权即可访问授权业务资源，不受所处物理位置限制，也不局限于业务资源的实际部署位置。

  - 每个业务资源被两个甚至多个TMG代理，终端用户发起访问时，TMA会根据链路实际状况智能选择最优路径进行数据传输，提升了网络的高可用性，变相扩展了业务访问带宽，为终端用户提供最佳访问体验。

  - 与公司已建的SSO系统打通，员工在TMA上经认证授权登录 TM 后，即可直接访问所有授权业务资源，无需每访问一个业务资源认证一次。

5. **对所有业务访问行为进行审计以及异常检测告警与处置**

  - TM 实时采集全网每一个终端的系统环境数据以及业务访问流量，并对所有数据打上身份标签。当遇到安全事件时，可基于身份化的历史数据追溯到具体的用户和终端。

  - 管理员可以集中管理全网终端，可随时查询全网终端信息，例如：通过进程HASH值，查询全网有多少终端运行过同样的进程，或通过补丁编号，查询全网有哪些终端打过和没打过这个补丁等。

  - 管理员还可以针对含有敏感数据的业务资源开启强制终端用户必须使用TMA自带的安全浏览器进行访问，启用所有页面打上水印、禁止下载等限制策略，并跟踪监控每一个用户都访问查看了哪些敏感数据。

  - TM 通过AI技术基于身份化的终端和网络行为数据进行用户行为建模，掌握每个用户每个终端的状态，生成行为基线。管理员可设置策略，当用户行为出现偏离基线的操作时强制进行二次验证，也可直接禁用帐号或终端。
