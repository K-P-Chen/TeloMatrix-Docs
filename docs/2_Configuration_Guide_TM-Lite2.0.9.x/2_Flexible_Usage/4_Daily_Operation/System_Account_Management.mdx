TM 安全云网存在多个不同类型的系统管理帐号，分别用于在不同场景下对 TM 各组件和策略进行集中配置和管理。

## TM 企业帐号

TM 企业帐号是您在 TM 企业注册平台为企业注册的 TM 服务帐号。在完成企业实名认证后，TM 企业帐号与企业统一社会信用代码绑定。

未完成企业实名认证的 TM 企业帐号只可享有1个月 TMLite版本的免费试用权限，完成企业实名认证的 TM 企业帐号可享有永久免费的 TMLite版本使用权限，以及在付费后享有 TMPro使用权限。

TM 企业帐号在您登录 TM 企业注册平台，以及安装 TM 组件后进行证书同步激活组件时使用。

## TM 组件CLI控制台帐号

所有 TM 组件，包括：TMC、TMG、TMD、TMI、TME，在服务器或虚拟机上部署成功后，管理员可直接通过控制台帐号登陆 TM 组件的CLI控制台界面，或者通过SSH客户端并使用控制台帐号登陆 TM 组件的CLI控制台界面，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Account_Management/6.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

TM 组件的CLI控制台是为了帮助管理员在没有网络的情况下为 TM 组件配置基础的网络信息、同步授权证书、启停Web管理控制台的超级管理员帐号、修改超级管理员帐号密码等操作。管理员也可以通过SSH客户端远程接入到 TM 组件的CLI控制台进行配置。

每一个 TM 组件只有一个CLI控制台帐号，且所有组件的初始帐号和密码都一致，因此在部署好 TM 组件后，您需要尽快修改默认密码并妥善保存此密码。

TMv209x版所有组件的控制台**用户名为：admin，默认密码为：b4a5928c**

## TMC Web管理控制台帐号

TMG、TMD、TME只具备CLI控制台界面，TMC、TMI除了CLI控制台界面以外，还具备Web管理控制台界面。Web管理控制台界面用于进行复杂的配置和操作，管理员可通过TMC集中对TMG、TMD、TME进行状态监控和基础配置。

Web管理控制台具备四种角色的系统帐号，支持系统管理权、业务配置权、操作审计权三权分立，角色定义、权限和配置方法如下（以TMC举例）：

### 超级管理员帐号

- **角色定义**

  超级管理员（sysadmin）具备Web管理控制台的最高权限，可用于配置TMC Web管理控制台的其他角色系统帐号。在TMC安装部署并更新证书后，生成超级管理员帐号初始密码，并将初始化邮件发送到 TM 企业帐号注册邮箱中。管理员可通过初始化邮件里的sysadmin帐号登录TMC的Web管理控制台。

- **帐号权限**

  具备TMC Web管理控制台的最高权限，可在TMC的Web管理控制台操作所有功能。

  超级管理员帐号只有一个，不可新建其他超级管理员角色帐号，可修改自身密码，可禁用超级管理员帐号（可在CLI控制台重新启用）。

- **配置方法**

  - **修改密码**

    选择菜单【系统管理】->【系统帐号】打开系统帐号配置页面，如下图所示：

  <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Account_Management/1.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

   在页面上方的超级管理员帐号配置区，点击“修改密码”按钮，打开sysadmin帐号密码修改抽屉页，如下图所示：

  <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Account_Management/2.png" style={{width: "40%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />
    
   填写sysadmin原密码，输入新密码并再次确认输入的新密码后，点击“保存”完成sysadmin密码的修改。需要注意，sysadmin的密码只有sysadmin自己可以修改，其他帐号无权限修改sysadmin的密码。

   如果您忘记了sysadmin密码，您可以登陆到CLI控制台界面重置TMC的sysadmin帐号。

  - **启停超级管理员帐号**

    在Web管理控制台禁用超级管理员帐号

    在页面上方的超级管理员帐号配置区，点击“关闭帐号”按钮即可停用sysadmin帐号，如下图所示：

   <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Account_Management/3.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />
     
    在CLI管理控制台启用超级管理员帐号

    sysadmin帐号被停用后，只能在TMC的CLI管理控制台重新启用，如下图所示：
    
   <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Account_Management/7.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />    

### 系统管理员帐号

- **角色定义**

  系统管理员（默认帐号：admin）具备Web管理控制台的系统运维管理权限，可用于配置TMC Web管理控制台的系统级配置，如：网络配置等。

  在TMC安装部署并更新证书后，TMC生成系统默认的系统管理员帐号初始密码，并将初始化邮件发送到 TM 企业帐号注册邮箱中。系统管理员可通过初始化邮件里的admin帐号登录TMC的Web管理控制台。

- **帐号权限**

  具备TMC Web管理控制台的系统管理权限，可在TMC的Web管理控制台操作所有跟TMC以及被接管的 TM 组件的所有系统管理功能，包括：
  
  - 监控 TM 安全云网服务授权有效期
  
  - 通过云网监控功能集中管理 TM 安全云网中的所有 TM 组件
  
  - 配置系统时间、系统接口、告警阈值、设置云网地址池
  
  - 配置自定义门户，包括背景图、LOGO、版权信息等
  
  - 可配置系统管理员帐号，但不包括超级管理员帐号

  - 可执行服务重启、系统重启、恢复出厂设置、系统关机等操作
  
  - 可导出系统调试日志、客户端调试日志

  - 可对系统执行升级操作、指定TMA客户端升级操作

### 审计管理员帐号

- **角色定义**

  审计管理员（默认帐号：auditor）具备Web管理控制台的系统审计权限，可用于查看TMC Web管理控制台的各类日志。

  在TMC安装部署并更新证书后，TMC生成默认的审计管理员帐号初始密码，并将初始化邮件发送到 TM 企业帐号注册邮箱中。

  审计管理员可通过初始化邮件里的auditor帐号登录TMC的Web管理控制台。

- **帐号权限**
  
  具备TMC Web管理控制台的审计管理权限，包括：

  - 可查看终端日志、导出终端日志。
  
  - 可查看系统操作日志，导出和删除系统操作日志。
  
  - 可查看云网各组件的系统告警日志，导出和删除系统告警日志，可手动同步各组件日志。

### 业务管理员帐号

- **角色定义**

  业务操作员（默认帐号：operator）具备Web管理控制台的业务操作权限，可用于配置TMC Web管理控制台的业务相关配置，如：创建资源、创建策略等。

  在TMC安装部署并更新证书后，TMC生成业务操作员帐号初始密码，并将初始化邮件发送到 TM 企业帐号的注册邮箱中。

  业务操作员可通过初始化邮件里的operator帐号登录TMC的Web管理控制台。

- **帐号权限**

  具备TMC Web管理控制台的此业务操作权限，包括：

  - 可监控 TM 安全云网在线帐号、终端和业务资源，可强制下线在线的帐号和终端。
  
  - 可审批终端注册申请、可授权执行动态策略、可查看策略执行任务。
  
  - 可在授权范围内配置帐号资源、终端资源、业务资源、服务对象、证书管理。
  
  - 可配置认证策略、终端策略、授权策略、访问控制策略、API策略、链路优化策略。

  - 可配置内置安全浏览器策略、设置全局策略。

#### 创建系统帐号

选择菜单【系统管理】->【系统帐号】打开系统帐号配置页面，在页面下方的管理员列表功能区点击“新增”打开新增系统帐号页面，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Account_Management/4.png" style={{width: "60%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

**配置项**

**帐号：**新建的帐号，帐号名称命名要求不超过100个字符，只能是大小写字母、数字、下划线、减号、@、半角点号的组合，必填项。

**开启双因素认证：**可以为帐号开启强制的双因素认证，开启后您可选择使用 TMTOTP一次性口令，或者使用短信认证两种方式。
如果使用TOTP一次性口令，您可以在系统帐号列表的操作列点击“TOTP”查看种子文件二维码。您可以通过 TM 手机令牌APP，或者其他支持TOTP的手机令牌APP扫码此二维码获得系统帐号的一次性口令。

**短信网关：**如果选择短信认证方式，您还需要选择对应使用哪个已配置好的短信网关。

**姓名：**帐号姓名，要求不超过100个字符，只能是大小写字母、数字、下划线、减号、@、半角点号的组合，帐号姓名不可重复，必填项。

**手机号：**填写管理员真实可用的手机号，用于接收双因素认证口令，手机号不可重复，必填项。

**角色：**选择帐号角色，可选项包括：审计管理员、业务操作员、系统管理员。

**超时时间：**设置当管理员多长时间无操作后，Web管理控制台自动让管理员下线。

**尝试登陆次数：**设置当管理员登录失败几次后，帐号将被自动停用，以避免恶意登录。帐号停用后需要通过其他系统管理员或超级管理员重新启用。

**电子邮箱：**填写管理员真实可用的邮箱地址，用于接收帐号初始化邮件，邮箱不可重复，必填项。

**生效日期：**帐号从什么日期和时间开始启用。

**失效日期：**帐号到什么日期和时间停用。

**备注：**填写帐号的备注信息，限制最多200个字符。

**管理员访问控制：**限制管理员的访问源IP地址，默认为不限制，可设置允许访问的源IP地址范围，或禁止访问的IP地址范围，支持同时设置多个IP网段。

#### 业务管理员分级授权

在分布式企业当中，业务资源通常分布在多个云、多个数据中心，企业内每个业务部门经常会对业务资源进行调整，弹性扩容或上线新的业务。如果每一次的调整都需要网络管理员或安全管理员进行操作，则会给网络管理员或安全管理员增加大量的工作负担，同时还会影响业务上线或优化的工作效率。

TM 安全云网支持分级的业务管理员授权，即将部分网络资源和可授权帐号范围授权给业务部门的管理员，由业务部门管理员自行发布自己部门的业务资源，并给自己部门的员工进行授权。网络管理员或安全管理员无需参与到业务部门内部的业务发布和授权管理，仅需为业务部门配置好授权范围，监控业务部门管理员的授权策略和实际应用效果即可。

业务部门操作员授权方法如下：

1. 选择菜单【系统管理】->【系统帐号】打开系统帐号配置页面，在页面下方的管理员列表功能区点击“新增”打开新增系统帐号页面，创建一个业务操作员帐号。业务操作员在没有进行精细化授权之前默认是所有范围授权，可发布所有网络范围的业务资源和授权给企业内所有帐号和终端。

2. 在系统帐号列表中选中刚才新建的业务操作员帐号，点击“操作”列里的“授权限制”按钮，打开业务操作员的授权范围编辑页面，对业务操作员进行精细化授权，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Account_Management/5.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

3. 选择可以为哪些帐号进行授权，在帐号组筛选条件中勾选需要授权的帐号组，勾选后右侧已选条件框内会显示已勾选的帐号组。

4. 确认是业务管理员对本地帐号组是只读权限还是读写权限。只读权限指业务管理员只可对帐号组进行业务资源的授权，不可新增、编辑或删除帐号，读写权限意指业务管理员可以在授权帐号组内的新增、编辑和删除帐号。（只有本地帐号可增删改，第三方身份源帐号都是只读权限）

5. 选择授权发布网关，即授权业务管理员能使用哪些已部署好的TMG、TME发布自己的业务资源。

6. 勾选是否允许业务管理员发布域名资源。域名资源不受IP地址范围的限制，只要企业的DNS可以解析的域名都可以发布。

7. 选择发布业务资源的IP地址范围。您可以给业务管理员分配多个网络地址段，支持CIDR格式掩码。

8. 点击“确定”按钮，完成对业务操作员的精细化授权。
