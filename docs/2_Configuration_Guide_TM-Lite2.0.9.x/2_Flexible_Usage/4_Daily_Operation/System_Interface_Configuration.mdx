## 邮件服务器配置

TM 安全云网在日常运维过程中经常使用到邮件服务器，例如：帐号初始化邮件发送、帐号密码找回、修改帐号信息、告警邮件发送等。

因此，**请务必在安装部署好TMC后，尽早配置好邮件服务器，以免影响正常使用**。

选择菜单【系统管理】->【基础配置】打开系统基础配置页面，点击“系统接口”标签页，打开接口配置页面。

在页面左侧接口类型选择区域，选择“邮箱配置”，右侧显示邮箱服务器配置，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/1.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

**配置项**

**服务器地址：**填写您企业邮件服务器的发件服务（SMTP协议）IP地址或域名。

**服务器端口：**填写您企业邮件服务器的发件服务端口号，默认为25。

**发件人姓名：**您需要指定一个您企业邮箱服务里的一个发件人姓名。

**用户名：**填写此发件人邮箱帐号。

**密码：**填写此发件人邮箱帐号的密码。

**SSL：**是否启用SSL加密通信协议。

**SSL协议端口：**填写SSL加密后的SMTP服务端口号，默认为465。

在配置好以上配置项后，点击“保存”按钮，即可完成邮件服务器的配置。

您可以再下方邮箱测试区域填写一个收件地址，点击“测试”按钮，TMC将通过配置好的邮箱服务器，以刚才配置的发件人的身份向您填写的收件地址发送测试邮件。如果您能正常收到测试邮件，则说明邮件服务器配置成功。

## SYSLOG接口配置

选择菜单【系统管理】->【基础配置】打开系统基础配置页面，点击“系统接口”标签页，打开接口配置页面。在页面左侧接口类型选择区域，选择“SYSLOG接口”，右侧显示SYSLOG服务器配置，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/2.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />


SYSLOG接口配置非常简单，您只需填写SYSLOG服务器的IP地址或域名，以及SYSLOG服务端口，点击“保存”按钮即可。

配置好SYSLOG服务器接口后，TMC将实时地发送TMC以及其他 TM 组件的系统告警到指定的SYSLOG服务器。

## SNMP接口配置

TMC支持提供SNMP Agent服务，可开放管理接口给网络监控系统，用于日常对TMC的状态监控。

选择菜单【系统管理】->【基础配置】打开系统基础配置页面，点击“系统接口”标签页，打开接口配置页面。在页面左侧接口类型选择区域，选择“SNMP设置”，右侧显示SNMP服务器配置，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/3.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />


**配置项：**

**SNMP：**开启或关闭SNMP功能，选中“启用”则意味开启SNMP Agent服务，选中“停用”则意味着停止SNMP Agent服务。

**端口：**填写SNMP Agent的服务端口，可配置端口包括161，或1024-65535范围之内的端口号。

**SNMP版本：**选择SNMP的支持版本，不同的版本配置的配置项会有不同，可选项包括：SNMPv1&v2c、SNMPv3。

1. 当选择SNMPv1&v2c时，配置如下：
   a. SNMP访问限制配置为允许任意来源地址
  
  <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/4.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

   团体（community）是SNMP基本的安全机制，用于实现SNMP网络管理员访问SNMP管理代理时的身份验证，类似特权访问密码。
    
   **只读团体字符串：**输入只读属性的团体字符串。

   **读写团体字符串：**输入读写属性的团体字符串。

   b. SNMP访问限制配置为仅指定地址

   <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/5.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

   当设置为仅指定地址时，意味着TMC只接受来自指定源IP地址的访问，需要在配置中指明只读团体的源IP地址限制范围和读写团体的源IP地址限制范围。

2. 当选择SNMPv3时，配置如下：

  <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/6.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

  **配置项**

  **用户名：**配置SNMPv3用户名。

  **安全等级：**安全等级分为三种，分别是authPriv（鉴权且加密）、authNoPriv（只鉴权不加密）、noAuthNoPriv（不鉴权不加密）。

  **认证协议：**选择认证协议，可选项包括：MD5、SHA。

  **认证密码：**配置SNMPv3用户认证密码。

  **保密协议：**选择加密协议，可选项包括：DES、AES。

  **私有密码：**配置SNMPv3用户加密密码。

完成所有SNMP配置后，点击“保存”按钮，完成SNMP接口配置。


## 短信接口配置

TMC在日常运维过程中经常需要使用手机短信与管理员或终端用户进行交互，如：使用手机短信进行多因素认证、修改帐号信息时进行再次身份校验、帐号密码找回等。因此，**请务必在安装部署好TMC后，尽早配置好短信接口服务，以免影响正常使用**。

选择菜单【系统管理】->【基础配置】打开系统基础配置页面，点击“系统接口”标签页，打开接口配置页面。在页面左侧接口类型选择区域，选择“短信接口”，右侧显示短信接口列表，如下图所示：

  <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/7.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

TMC支持同时配置多个短信接口，您可在不同使用场景，调用不同的短信接口发送消息。
点击“新增”按钮，打开新增短信接口配置页面，TMC当前支持配置阿里云和腾讯云两种短信网关，配置方式分别如下：

1. 阿里云短信网关配置

 <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/8.png" style={{width: "60%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

   a. 基本信息
    
   接口名称：为短信接口命名，以方便后续引用，限制不超过30个字符，只能是中英文、数字、英文点组合，必填项。
    
   短信网关平台：选择阿里云。

   短信网关地址：阿里云的短信网关默认为dysmsapi.aliyuncs.com，此处无需修改，必填项。
 
   AccessKey ID/AccessKey Secret/短信签名内容：需要从阿里云短信网关配置中获取，并填入到此处，必填项。

   发送频率限制：可限制每小时或每天，最多发送多少条短信，非必填项。

   b. 短信模板

   您需要在阿里云的短信网关上新建短信模板，可以参考页面中提示的模板内容来创建（包括以下场景下的短信模板：找回密码验证码、修改绑定邮箱验证码、修改绑定手机号验证码、登录确认验证码）。

   在创建好短信模板后，您需要把阿里云的短信网关上生成的对应模板ID填入到此处。

   c. 短信测试

   填写完上述配置后，您可以在此处填写您的手机号，点击“测试”按钮，测试是否能正常通过以上配置正常发送短信到您的手机。

2. 腾讯云短信网关配置

    腾讯云短信网关配置页面如下图所示：

  <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/9.png" style={{width: "60%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />
    
   a. 基本信息
    
   接口名称：为短信接口命名，以方便后续引用，限制不超过30个字符，只能是中英文、数字、英文点组合，必填项。

   短信网关平台：选择腾讯云。

   短信网关地址：腾讯云的短信网关默认为sms.tencentcloudapi.com，此处无需修改，必填项。

   短信网关地域/SecretId/SecretKey/短信SdkAppId/短信扩展号/SenderId/短信签名内容：需要从阿里云短信网关配置中获取，并填入到此处，必填项。

   发送频率限制：可限制每小时或每天，最多发送多少条短信，非必填项。

   b. 短信模板

   您需要在阿里云的短信网关上新建短信模板，可以参考页面中提示的模板内容来创建（包括以下场景下的短信模板：找回密码验证码、修改绑定邮箱验证码、修改绑定手机号验证码、登录确认验证码）。
  
   在创建好短信模板后，您需要把阿里云的短信网关上生成的对应模板ID填入到此处。

   填写完上述配置后，您可以在此处填写您的手机号，点击“测试”按钮，测试是否能正常通过以上配置正常发送短信到您的手机。

## RADIUS接口配置

TMC支持通过Radius服务生成一次性密码，并作为终端用户登录时的多因素认证条件。

选择菜单【系统管理】->【基础配置】打开系统基础配置页面，点击“系统接口”标签页，打开接口配置页面。在页面左侧接口类型选择区域，选择“RADIUS接口”，右侧显示RADIUS接口列表，如下图所示：

  <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/10.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

TMC支持同时配置多个RADIUS接口，您可在不同使用场景，调用不同的RADIUS接口发送一次性口令。点击“新增”按钮，打开新增RADIUS接口配置页面，如下图所示：

  <img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/System_Interface_Configuration/11.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

**配置项**

**接口名称：**为RADIUS接口命名，以便后续引用，限制不超过30个字符，只能是中英文、数字、英文点组合，必填项。

**服务器地址：**填写RADIUS服务器IP地址，必填项。

**共享秘钥：**填写与RADIUS服务器交互报文时使用的共享秘钥，必填项。

**认证端口：**填写RADIUS服务器的UDP认证端口，默认为1812，必填项。

**认证超时时间：**可等待 RADIUS 服务器响应的秒数，默认为 10 秒，必填项。

**重试次数：**可重试失败请求的最大次数，默认为 3 次，必填项。

**认证协议：**TMC目前支持PAP、CHAP认证协议，二选一。

**密码拼接：**开启后，TMC会把终端用户登陆TMA时使用的密码与RADIUS口令拼接在一起，然后提交到RADIUS服务器进行认证。默认关闭，此功能用于一些特殊的RADIUS认证系统。
