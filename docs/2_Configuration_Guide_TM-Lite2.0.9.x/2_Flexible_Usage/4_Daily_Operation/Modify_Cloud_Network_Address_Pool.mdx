## 功能说明

TM 安全云网将分布在不同物理位置的终端和业务资源整合在一张虚拟网络中，可能出现物理网络与虚拟网络 IP 地址冲突。部署前需了解业务资源所在网络的 IP 地址范围，通过配置云网地址池避免冲突。

## 配置方法

1. 选择菜单【系统管理】->【基础配置】->【云网地址池】打开配置页面：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/Modify_Cloud_Network_Address_Pool/1.png" style={{width: "50%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

2. 填写云网地址池网段范围，支持 CIDR 掩码。TM 支持主备双网段配置：当终端或业务资源主网段 IP 与本地物理网络在同一网段时，自动更换为备网段 IP。

3. 点击"保存"按钮，在弹出的提示中点击"确定"，云网地址将被重置：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/Modify_Cloud_Network_Address_Pool/2.png" style={{width: "40%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

修改云网地址池会重置整个 TM 安全云网，所有组件虚拟网地址和网络配置都将改变。请在无终端用户访问时操作，操作完成后检查每个组件网络在线状态。
