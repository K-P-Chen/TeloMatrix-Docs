## 功能说明

当您在企业网络里部署了 TM 安全云网，理论上您已经构建了一张拥有完整IPv4地址空间的私有虚拟网络。

但由于 TM 安全云网是将位于任意位置的终端和业务资源整合在一张虚拟网络中，而每个终端和业务资源所处的物理网络环境十分复杂，容易出现物理网络与虚拟网络IP地址冲突的现象，导致网络访问受到影响。

因此，我们建议您在部署 TM 安全云网前，了解清楚您的业务资源所处网络都分配了哪些IP地址，在部署 TM 安全云网时通过配置云网地址池，尽量让虚拟网络的IP地址范围不要与业务资源在同样的IP地址范围内。

## 配置方法

1. 选择菜单【系统管理】->【基础配置】->【云网地址池】打开云网地址池配置页面，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/Modify_Cloud_Network_Address_Pool/1.png" style={{width: "50%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

2. 填写云网地址池的网段范围，支持扩展掩码（CIDR无分类编址掩码）。

  为了尽量避免虚拟网IP地址与终端或业务资源的IP地址范围出现冲突，TM 支持主备双网段配置，即当发现终端或业务资源的主网段IP地址与本地物理网络在同一个网段的时候，自动更换为备业务网段IP地址。

3. 点击“保存”按钮，页面返回下图所示的提示，点击“确定”TM 安全云网的网络地址将被重置。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/4_Daily_Operation/Modify_Cloud_Network_Address_Pool/2.png" style={{width: "40%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />
  
  修改云网地址池会重置整个 TM 安全云网，所有 TM 组件的虚拟网地址都将改变，所有组件的网络配置都将自动重置。

  因此，请务必在无终端用户访问业务的时候操作，并且在操作完成后检查每一个组件的网络在线状态，以确保网络重置后运行正常。
