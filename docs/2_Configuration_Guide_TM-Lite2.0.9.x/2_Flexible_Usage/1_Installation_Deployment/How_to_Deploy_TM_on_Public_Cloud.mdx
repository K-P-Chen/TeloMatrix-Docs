title: "如何在公有云上部署 TM"

---

> TM 安全云网的各个组件都可以部署在公有云环境中，支持在多云、混合云环境中部署，将企业的碎片化网络重新整合在一张安全的虚拟网络。
>
> 本文以在阿里云上安装部署 TM 组件为例，描述如何在公有云环境中部署 TM 安全云网。

## TM 组件部署拓扑

在云环境中部署 TM 安全云网的常见方式如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/1.png" style={{width: "80%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

- 如果企业的业务资源部署在公有云中，您希望将公有云里的业务资源发布到 TM 安全云网，方便终端用户在任意位置访问。您只需要在业务资源所在的安全组内再申请一台虚拟机，将TMG镜像导入到虚拟机中安装即可，申请的虚拟机无需额外的弹性公网IP和互联网带宽，直接复用VPC的公弹性公网IP和带宽即可，如图中右侧的安全组所示。

- 如果您希望将TMC、TMD等 TM 组件也部署在公有云里，建议您在VPC中单独建立一个安全组，将TMC和TMD部署在此安全组中。同样，TMC和TMD也无需额外的弹性公网IP和互联网带宽，直接复用VPC的公弹性公网IP和带宽即可，如图中右侧的安全组所示。

- 您需要为部署不同 TM 组件的虚拟机开通相应的网络访问权限，关于网络权限的设置请参考：[物理网络访问权限要求](../Installation_Deployment/Physical_Network_Access_Permission_Requirements)。

## 在阿里云部署 TM 组件

### 选购 TM 组件

在阿里云云市场的安全市场中搜索“TM”，您可以搜到 TMLite的TMC、TMG、TMD三个组件，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/2.png" style={{width: "80%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

点击查看详情打开产品详情和购买页，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/3.png" style={{width: "80%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

### 基础配置

点击“立即购买”即可进入自定义购买页面，在页面中选择购买实例的基础配置，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/4.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/5.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

付费模式，地域以及可用区，可根据您的规划自行选择。实例规格需要选择x86架构，您可根据不同的需求选择相应的分类，推荐使用“通用型”和“计算型”。

如想了解不同类型的云主机特点，可查阅[阿里云相关说明文档](https://help.aliyun.com/document_detail/108375.html)。

不同的虚拟机配置可以承载的网络规模和访问流量会有不同，具体的配置建议请参考：[服务器配置建议](../Installation_Deployment/Server_Configuration_Recommendations)。这里为了示范，选择最小配置为：4vCPU、8GiB内存、100GiB硬盘。

在镜像选择区，确认当前选择的镜像为刚才在云市场上购买的 TM 组件镜像，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/6.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

如果您不是从云市场直接点击购买镜像进入的自定义购买页面，您需要在镜像选择配置区，选择“镜像市场”，点击“从镜像市场获取更多选择（含操作系统）”，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/7.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

点击后，在镜像搜索页面的搜索条件输入框输入“TM 安全云网”，你可以搜索到 TM 相关组件，点击“使用”可将对应的 TM 组件镜像作为虚拟机的安装镜像，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/8.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

### 网络与安全组配置

点击下一步，进入“网络与安全组配置”页面，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/10.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

选择 TM 组件计划部署的位置，包括：私有网络（VPC）、虚拟交换机。

注意：所选择的VPC和虚拟交换机需要具备公网NAT网关，以便 TM 组件可以通过SNAT方式与互联网通信，同时可以通DNAT方式配置TMG互联网映射IP地址和端口，以及让管理员可以远程登录 TM 组件的WEB控制台或CLI控制台管理配置 TM 组件。

如VPC网络已与本地管理员所在网络打通，则可以忽略DNAT方式远程访问 TM 组件的WEB控制台与CLI控制台。如果您的云上资源没有对VPC网络进行规划，也可以使用分配公网IPv4地址方式进行远程云主机的管理与使用（出于安全性考虑，不建议这种方式）。本例主要以VPC方式进行配置讲解。

点击选择安全组，为创建的虚拟机配置网络访问策略，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/11.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

注意：我们建议安全组的组内联通策略，使用“组内互通”，如果您的安全组使用的是“组内隔离”则请参考[物理网络访问权限要求](../Installation_Deployment/Physical_Network_Access_Permission_Requirements)。

关于阿里云安全组的使用说明，请参考[阿里云官方文档](https://help.aliyun.com/document_detail/25467.html)。

### VPC网络配置

在云服务器ECS控制台，进入VPC管理页面，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/15.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

选择【NAT网关】->【公网NAT网关】，进入公网NAT网关配置页面。（如所使用的VPC中没有配置公网NAT网关，则需要创建，具体操作过程请参考[阿里云官方文档](https://help.aliyun.com/document_detail/194181.html?spm=5176.11182189.help.dexternal.44e74882DEiRuV)。）

首先设置SNAT，使组件可与互联网进行通信。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/16.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

配置SNAT如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/17.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

阿里云提供了四种SNAT粒度，这里使用自定义网段粒度来做演示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/18.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/20.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/How_to_Deploy_TM_on_Public_Cloud/21.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

以上完成了TMC组件的安装以及相关环境的配置，其他组件的操作与其类似，相关网络权限设置请参考[物理网络访问权限要求](../Installation_Deployment/Physical_Network_Access_Permission_Requirements)。
