企业的Web业务资源可以通过代理模式或云网路由模式发布到 TM 安全云网，这两种不同的模式要求终端用户分别使用云网IP地址或者内网物理IP地址访问业务资源。

当终端用户使用域名访问这两种部署模式下的业务资源时，同样的域名需要解析成不同的IP地址。TM 提供了灵活的域名解析方案，可以满足在各种类型的网络环境中实现最优的域名解析效果。

## TM 域名解析方式

### 使用 TMDNS服务（TMD）解析（推荐方式）

TMDNS（TMD）记录了 TM 安全云网中所有通过域名进行代理的业务资源的域名解析记录，是 TM 专门用来解析云网内域名业务资源的核心组件。

当管理员使用域名代理业务资源时，TMC会自动在TMD上配置该域名的解析记录，解析记录为代理此业务资源的一个或多个TMG云网IP地址。同时TMD将该解析记录推送到对应的TMG，由TMG为用户终端提供首选的DNS解析服务，而TMD作为TMG的递归DNS。

当终端用户登录TMA后，将随机获取TMG的云网IP地址作为本地终端的DNS解析地址。此时，终端用户的所有DNS请求都会发送给此TMG处理，如命中TMG自身已有的域名解析记录则返回解析结果，如没有命中，则将请求交给TMD处理。

注意：TMD自身的DNS需要配置为企业内部DNS服务器IP地址，当TMD无法解析时TMD会继续将域名解析请求交给内网DNS处理，这样可以最大程度的满足用户对内网资源的访问，可同时兼容域名代理模式和路由模式的业务资源。

### 使用终端本地HOST文件解析

管理员可以通过配置认证模板，指定部分帐号在登陆TMA后将已授权的、通过域名代理的业务资源域名解析记录写入到终端本地的HOST文件中。当终端用户使用域名访问发布到 TM 安全云网的业务资源时，直接通过本地HOST文件进行解析。终端用户同时访问其他互联网资源的域名，则通过终端本地配置的DNS服务器进行域名解析。

注意：HOST解析方式适合终端与企业内部DNS位于不同区域，且终端在访问业务资源的同时还需要高效地访问互联网资源的场景。

由于互联网的特性，位于不同区域的终端在访问同一个域名的时候，通过本地运营商分配的DNS解析出来的结果可能不同，因此当终端跨域地通过企业内部DNS解析互联网资源域名，其解析结果在终端本地访问速度会相对较慢，甚至无法打开。

使用HOST解析方式，可以让终端依然通过本地运营商分配的DNS解析互联网资源，但发布到 TM 的企业内部业务资源可以通过本地HOST进行解析。

使用终端HOST文件进行解析的局限是：

1. 此方式仅适用于PC端和Mac端。

2. 通过HOST文件无法解析使用云网路由模式发布的业务资源，云网路由发布的业务资源需要通过企业内部DNS进行解析。

### 使用企业内网或外网DNS解析

管理员可以在TMC上发布企业自己的内网DNS服务资源或外网DNS服务资源，并将此资源授权给指定帐号和终端。被授权的帐号终端在登录 TM 安全云网后，终端本地DNS将被配置为企业的DNS地址，终端的所有域名解析请求都将使用企业DNS进行解析。

注意：使用企业自身DNS进行域名解析的方式只适合对云网路由模式发布的业务资源进行域名解析（因为在云网路由模式下，终端用户实际访问的IP地址为业务资源的内网IP地址）。如果需要通过企业DNS解析域名代理模式发布的业务资源，则需要修改企业DNS的解析记录，将原解析记录里的业务资源物理IP地址改为业务资源的云网IP地址。

## 域名解析应用场景举例

### TMD解析（推荐方式）

#### 使用场景

如果企业的多个业务资源同时通过域名代理模式以及路由模式发布到 TM 安全云网，并且业务资源分布在多个不同地域由多个TMG代理，建议您采用TMD解析的方式。

部署方案如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/1.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

在此模式中，当终端用户接入到 TM 安全云网，终端的云网DNS地址将被随机配置成某个TMG的云网地址（如图中的10.10.0.102）。

当终端用户使用域名访问业务资源时（`www.crmtest.com`）
，首先会通过TMG(10.10.0.102)进行域名解析，如果业务资源是由此TMG通过域名进行的代理，则直接向终端返回解析结果。

如果业务资源不是由此TMG通过域名进行的代理，则TMG会向TMD(10.10.0.104)进行递归查询。

TMD为 TM 安全云网中的权威DNS服务器，可以解析 TM 安全云网中通过域名代理的所有业务资源。

如果业务资源通过路由模式进行代理（如：`www.oatest.com`），
则域名解析记录仅存在企业的内网DNS上，此时TMD会继续向内网DNS进行递归查询，然后将解析结果返回给终端。

#### 配置方法

1. 在 TM 安全云网中部署TMD，并将TMD的本地DNS配置成企业内网DNS的IP地址。

2. 在认证模板的域名解析选项中选择“使用DNS解析”。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/2.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

3. 将认证模板应用到指定的帐号或帐号组。

#### 注意事项

1. 跨域DNS请求问题：在登录到 TM 后，终端在访问非 TM 安全云网中的域名时，会将域名解析请求转发到内网DNS上进行解析。如果终端与内网DNS位于不同的地域，终端得到的域名解析结果可能与终端使用本地物理网DNS解析的结果不一致，从而导致终端通过域名在本地访问非 TM 安全云网中的站点时速度变慢，甚至无法打开。

2. TM 默认使用HOST文件或TMD进行域名解析，但如果配置了内网DNS或公网DNS资源并授权给了终端用户，则内网DNS和公网DNS优先级高于使用HOST文件或TMD进行解析。因此在使用TMD进行域名解析时，请确认没有配置内网或外网DNS并授权给了终端用户。

### HOST域名解析

#### 使用场景

如果企业需要终端通过域名进行访问的业务资源全都是通过域名代理模式发布到 TM 安全云网上，则可以使用修改本地HOST文件的方式进行解析。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/3.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

如图所示，OA资源通过域名代理的方式接入到 TM 安全云网，远程终端在登录 TM 安全云网后，本地系统的HOST文件将被修改，增加本地解析记录“10.10.0.102    `www.oatest.com`”。

当终端用户打开浏览器访问`www.oatest.com`
时，首先通过本地HOST文件进行解析，得到返回结果为10.10.0.102，浏览器后续将向目标地址为10.10.0.102发起https的访问请求。

如果终端用户同时访问类似`www.baidu.com`
这样的公网域名，则通过终端本地由接入运营商分配的DNS进行解析。

#### 配置方法

1. 配置认证模板，在域名解析选项中选择“写入HOST文件”。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/4.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

2. 将认证模板应用到指定的帐号或帐号组，此帐号下的Windows、Mac客户端再次登录后将使用修改本地HOST方式进行域名解析。

#### 注意事项

1. 修改本地HOST文件的方式仅对Windows、Mac客户端有效，对iOS、Android客户端无效.

2. 修改本地HOST文件的方式仅对使用域名代理的Web应用资源有效，如果使用云网路由模式进行代理，则需要通过IP地址进行访问。

### 使用公网DNS解析

#### 使用场景

如果您的企业已经使用了部署在公网上的DNS实现对私有业务资源的域名解析，您可以直接使用您的公网DNS作为内部私有业务资源的域名解析，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/5.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

企业在公网部署了自己的公网DNS服务器（假设IP地址为：a.a.a.a），所有内网业务资源（如OA服务器，域名为：`www.tmtest.com`）
的域名可以通过公网DNS服务器解析成业务资源所在内网的IP地址（OA服务器IP地址为：172.16.10.13）。

用户从终端打开浏览器，访问`https://www.tmtest.com`
，终端会先去公网域名服务器a.a.a.a请求`www.tmtest.com`
的域名解析结果，公网域名服务器返回解析结果为172.16.10.13。

浏览器根据解析结果向IP地址为172.16.10.13的服务器发起https请求，并完成后续的业务访问。

注意：这里终端用户是从远端访问业务资源的局域网IP地址，因此业务资源必须是通过云网路由模式进行业务流量转发。

#### 配置方法

1. 在TMC上配置公网DNS资源（a.a.a.a），并将公网DNS资源授权给所有帐号和终端。

2. 当终端用户使用TMA登录 TM 安全云网时，本地终端的虚拟网卡DNS将被配置为a.a.a.a。

#### 注意事项

1. 无需在公网域名服务器上对业务资源的域名解析做任何调整，不需要对现有网络配置做任何改动。

2. 因为需要远程终端直接访问业务资源的局域网IP地址，因此只支持通过云网路由模式代理的业务资源。

### 使用内网DNS解析

#### 使用场景

当企业的业务资源都是以云网路由方式发布到 TM 安全云网时，可以使用此种DNS解析方式.

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/6.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

内网DNS域名解析，即终端用户使用公司内部的DNS作为本终端的域名解析地址。当终端用户使用域名访问业务资源时，将直接使用内网DNS服务进行域名解析。

#### 配置方法

1. 在TMC上配置内网DNS资源（如图中的172.16.10.11），指定内网DNS的代理TMG，并将内网DNS资源授权给所有帐号和终端。

2. 当终端用户使用TMA登录 TM 安全云网时，本地终端的虚拟网卡DNS将被配置为内网DNS服务器地址（如图中的172.16.10.11）。

#### 注意事项

1. 无需在内网域名服务器上对业务资源的域名解析做任何调整，不需要对现有网络配置做任何改动。

2. 使用内网DNS解析的方式只支持解析通过云网路由模式代理的业务资源。

### 使用TMD自定义模式解析

#### 使用场景

当企业需要终端使用域名访问其他发布到 TM 的非Web类的应用资源时，如：FTP、SMTP、SSH等，您可以在TMD上指定域名解析记录，像使用DNS服务器一样，在 TM 安全云网内通过TMD为所有终端提供指定的域名解析服务。

#### 配置方法
1. 选择菜单【状态监控】->【云网监控】，打开“TMDNS”标签页，在操作区点击“远程管理”打开TMD配置页面，选择“解析配置”打开TMD域名解析记录配置页面，如下图所示：

2. 点击“新增”按钮，打开新增域名解析记录配置页，如下图所示：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/7.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

3. 输入需要解析的域名和对应的解析结果IP地址，点击“保存”按钮，完成域名解析记录的新增。
