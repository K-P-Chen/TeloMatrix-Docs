title: "云网域名解析方案"

当终端用户使用域名访问代理模式或云网路由模式发布的业务资源时，同一域名需解析成不同的 IP 地址（云网 IP 地址或内网物理 IP 地址）。TM 提供以下域名解析方案。

## TM 域名解析方式

### 使用 TMDNS 服务（TMD）解析（推荐方式）

TMD 记录了 TM 安全云网中所有通过域名代理的业务资源的域名解析记录。

管理员使用域名代理业务资源时，TMC 自动在 TMD 上配置该域名的解析记录，解析结果为代理此业务资源的 TMG 云网 IP 地址。TMD 同时将该解析记录推送到对应 TMG，由 TMG 为终端提供首选 DNS 解析服务，TMD 作为 TMG 的递归 DNS。

终端用户登录 TMA 后，随机获取 TMG 的云网 IP 地址作为本地 DNS 解析地址。所有 DNS 请求发送到 TMG 处理：命中 TMG 已有记录则返回解析结果，未命中则交 TMD 处理。

注意：TMD 自身的 DNS 需配置为企业内部 DNS 服务器 IP 地址。TMD 无法解析时继续将请求交给内网 DNS 处理，这样可同时兼容域名代理模式和路由模式的业务资源。

### 使用终端本地 HOST 文件解析

管理员通过认证模板指定部分帐号在登录 TMA 后将已授权的域名代理业务资源解析记录写入终端本地 HOST 文件。终端使用域名访问时通过本地 HOST 解析，访问其他互联网资源时通过终端本地配置的 DNS 解析。

注意：HOST 解析方式适合终端与内网 DNS 位于不同区域，且需同时高效访问互联网资源的场景。由于不同区域终端通过企业内部 DNS 解析互联网域名结果可能与本地区域不同而导致访问速度慢，HOST 方式可让终端通过本地运营商 DNS 解析互联网资源，企业内部资源通过本地 HOST 解析。

局限：

1. 仅适用于 PC 端和 Mac 端。
2. 无法解析云网路由模式发布的业务资源（云网路由发布的资源需通过企业内部 DNS 解析）。

### 使用企业内网或外网 DNS 解析

管理员在 TMC 上发布企业内网 DNS 或外网 DNS 服务资源，授权给指定帐号和终端。被授权终端登录 TM 安全云网后，本地 DNS 被配置为企业 DNS 地址。

注意：企业 DNS 解析只适合对云网路由模式发布的业务资源进行解析（终端实际访问的 IP 地址为内网 IP 地址）。如需解析域名代理模式发布的业务资源，需修改企业 DNS 的解析记录，将原解析记录中的业务资源物理 IP 地址改为云网 IP 地址。

## 域名解析应用场景举例

### TMD 解析（推荐方式）

#### 使用场景

企业多个业务资源同时通过域名代理模式和路由模式发布到 TM 安全云网，且分布在多个地域由多个 TMG 代理时，建议使用 TMD 解析。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/1.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

终端用户接入 TM 安全云网后，云网 DNS 地址随机配置成某个 TMG 的云网地址（如图中 10.10.0.102）。终端用户使用域名访问业务资源时（如 `www.crmtest.com`），首先通过该 TMG 进行域名解析。如果业务资源由此 TMG 通过域名代理则直接返回解析结果，否则 TMG 向 TMD（10.10.0.104）递归查询。TMD 是 TM 安全云网中的权威 DNS 服务器，可解析所有通过域名代理的业务资源。

如果业务资源通过路由模式代理（如 `www.oatest.com`），域名解析记录仅存在于企业内网 DNS 上，TMD 继续向内网 DNS 递归查询后返回结果给终端。

#### 配置方法

1. 部署 TMD，将 TMD 的本地 DNS 配置成企业内网 DNS 的 IP 地址。
2. 在认证模板的域名解析选项中选择"使用 DNS 解析"。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/2.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

3. 将认证模板应用到指定帐号或帐号组。

#### 注意事项

1. 终端访问非 TM 安全云网中的域名时会将解析请求转发到内网 DNS。如果终端与内网 DNS 位于不同地域，解析结果可能与本地物理网 DNS 解析结果不一致，导致本地访问非 TM 安全云网站点时速度变慢或无法打开。

2. TM 默认使用 HOST 文件或 TMD 进行域名解析，但如果配置了内网 DNS 或公网 DNS 资源并授权给终端用户，内网 DNS 和公网 DNS 优先级高于 HOST 文件或 TMD 解析。使用 TMD 解析时，确认没有配置内网或外网 DNS 并授权给终端用户。

### HOST 域名解析

#### 使用场景

企业需通过域名访问的业务资源全部通过域名代理模式发布到 TM 安全云网时，可使用修改本地 HOST 文件方式解析。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/3.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

OA 资源通过域名代理接入 TM 安全云网，远程终端登录后本地 HOST 文件增加解析记录"10.10.0.102 `www.oatest.com`"。终端用户打开浏览器访问 `www.oatest.com` 时，首先通过本地 HOST 解析得到 10.10.0.102，浏览器向该地址发起 HTTPS 请求。同时访问公网域名如 `www.baidu.com` 时，通过终端本地运营商 DNS 解析。

#### 配置方法

1. 配置认证模板，域名解析选项选择"写入 HOST 文件"。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/4.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

2. 将认证模板应用到指定帐号或帐号组，Windows、Mac 客户端下次登录后使用 HOST 方式解析。

#### 注意事项

1. 仅对 Windows、Mac 客户端有效，iOS、Android 客户端无效。
2. 仅对域名代理的 Web 应用资源有效，云网路由模式需通过 IP 地址访问。

### 使用公网 DNS 解析

#### 使用场景

企业已使用部署在公网上的 DNS 对私有业务资源进行域名解析时，可直接使用公网 DNS。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/5.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

企业公网 DNS 服务器（IP a.a.a.a）将内网业务资源域名（如 `www.tmtest.com`）解析为业务资源所在内网 IP 地址（如 172.16.10.13）。用户从终端浏览器访问 `https://www.tmtest.com`，终端向公网 DNS 请求解析得到 172.16.10.13，浏览器向该 IP 发起 HTTPS 请求完成业务访问。

注意：终端用户从远端访问业务资源局域网 IP 地址，业务资源需通过云网路由模式进行流量转发。

#### 配置方法

1. 在 TMC 上配置公网 DNS 资源（a.a.a.a），授权给所有帐号和终端。
2. 终端用户使用 TMA 登录 TM 安全云网时，本地虚拟网卡 DNS 配置为 a.a.a.a。

#### 注意事项

1. 无需修改公网域名服务器上业务资源的域名解析，无需对现有网络配置做改动。
2. 只支持通过云网路由模式代理的业务资源。

### 使用内网 DNS 解析

#### 使用场景

企业业务资源全部以云网路由方式发布到 TM 安全云网时使用。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/6.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

终端用户使用公司内部 DNS 作为本终端域名解析地址，使用域名访问业务资源时直接通过内网 DNS 服务解析。

#### 配置方法

1. 在 TMC 上配置内网 DNS 资源（如图中 172.16.10.11），指定代理 TMG，授权给所有帐号和终端。
2. 终端用户使用 TMA 登录 TM 安全云网时，本地虚拟网卡 DNS 配置为内网 DNS 服务器地址（172.16.10.11）。

#### 注意事项

1. 无需修改内网域名服务器上业务资源的域名解析，无需对现有网络配置做改动。
2. 只支持解析通过云网路由模式代理的业务资源。

### 使用 TMD 自定义模式解析

#### 使用场景

企业需要终端使用域名访问非 Web 类应用资源（如 FTP、SMTP、SSH 等）时，可在 TMD 上指定域名解析记录，在 TM 安全云网内通过 TMD 为所有终端提供域名解析服务。

#### 配置方法

1. 选择【状态监控】->【云网监控】，打开"TMDNS"标签页，点击"远程管理"打开 TMD 配置页面，选择"解析配置"。
2. 点击"新增"按钮打开新增域名解析记录配置页。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/7.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

3. 输入需要解析的域名和对应的解析结果 IP 地址，点击"保存"。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Cloud_Network_Domain_Resolution_Solution/8.png" style={{width: "50%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />
