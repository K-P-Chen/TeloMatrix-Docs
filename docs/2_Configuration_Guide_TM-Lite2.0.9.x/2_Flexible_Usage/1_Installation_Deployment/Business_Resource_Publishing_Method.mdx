TM 支持 4 层反向代理、7 层反向代理和云网路由三种方式将业务资源发布到 TM 安全云网。可同时使用多种接入方式，同一业务资源也可由多种方式发布。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/1_Installation_Deployment/Business_Resource_Publishing_Method/1.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

- 4 层或 7 层代理模式下：终端用户通过 `https://10.10.0.101:20443` 访问远端云服务 Web 资源。10.10.0.101 是 TMG 虚拟网 IP 地址，20433 为 TMG 代理端口号，协议为 TCP（4 层）或 HTTPS（7 层）。
- 云网路由模式下：终端用户通过 `https://172.16.10.13` 直接访问数据中心内 Web 业务资源。172.16.10.13 是业务资源局域网 IP 地址，终端用户可在任意物理位置访问。

## 代理模式

支持网络 4 层或 7 层反向代理方式发布业务资源。

### 4 层代理接入

TMG 基于 TCP/UDP 传输层协议对业务资源进行反向代理，支持各类应用层协议的业务资源，如：MYSQL、VNC、SMB、SIP 或企业自研应用服务。

**优势：**

1. 支持将所有基于 TCP/UDP 传输层协议的应用服务发布到 TM 安全云网。
2. 支持基于身份的精准网络访问控制授权，支持通过流量日志进行用户行为特征分析和异常研判。

**局限：**

1. 用户需在终端上使用虚拟网 IP 地址和端口访问业务，不能直接使用局域网物理 IP 地址和端口。
2. 默认情况下业务服务端收到的请求源 IP 地址均为 TMG 的内网 IP 地址。如通过 TMI 采集网络流量日志进行行为分析无须关心此问题，日志已包含帐号和终端身份信息。

### 7 层代理接入

#### Web 业务代理

支持通过 HTTP、HTTPS 应用层反向代理将 Web 业务资源发布到 TM 安全云网，支持 IP 端口代理和域名代理两种模式。

- **IP 端口代理**：终端用户使用 IP 地址和端口号作为 URL 访问业务资源。
- **域名代理**：终端用户使用域名访问。域名代理模式下，终端通过 TMD 或本地 HOST 对域名解析获取云网 IP 地址后访问。DNS 解析原理参考：[云网域名解析方案](Cloud_Network_Domain_Resolution_Solution)。

**优势：**

1. 对任意位置的 Web 业务资源执行基于帐号和终端的细粒度网络访问授权。
2. 可对 HTTPS 流量进行应用层协议解析，识别应用层协议字段并在附加身份信息后实时发送到 TMI（需导入 HTTPS 服务端 SSL 证书）。
3. 支持应用层访问控制，如 URI 访问授权。

**局限：**

1. IP 端口代理时，终端用户需修改业务资源的目标 IP 地址与端口号为虚拟网 IP 地址和端口号。
2. 业务服务端收到的请求源 IP 是 TMG 的内网 IP。如需获取每个终端实际 IP 地址，可通过 HTTP/HTTPS 的 XFF（X-Forwarded-For）字段获取。
3. 如 Web 业务资源存在 301 重定向、302 跳转或站外页面嵌套，且目标站点是企业内部业务资源，需将目标站点也发布到 TM 安全云网，并进行关联或放入同一资源组统一授权。配置方法参考：[页面跳转、重定向或嵌套处理方案](../2_Resource_Management/3_Business_Resource_Management/Page_Redirect_or_Embedding)。

#### SSH 代理

支持通过 SSH 应用层代理将主机资源接入 TM 安全云网。使用 TMA 接入 TM 时，终端用户可使用任意 SSH 客户端访问；使用 TME 接入时可通过 WebSSH 代理访问。

**优势：**

1. 对任意位置的主机资源执行基于帐号和终端的细粒度网络访问授权。
2. 支持应用层协议解析，识别执行指令、回显等 SSH 应用层字段，附加身份信息后实时发送到 TMI。
3. 支持应用层访问控制，如 SSH 指令集限制。

**局限：**

1. 终端用户使用 SSH 客户端时需改用虚拟网 IP 地址和端口访问，不能直接使用业务服务器局域网 IP 地址和端口。

#### RDP 代理

支持通过 RDP 应用层代理将 Windows 主机资源接入 TM 安全云网。使用 TMA 登录时可通过 Mstsc 客户端访问 Windows 桌面；使用 TME 登录时可通过 WebRDP 代理在浏览器中访问。

**优势：**

1. 对任意位置的 Windows 主机资源执行基于帐号和终端的细粒度网络访问授权。
2. 终端用户可通过 TME 用浏览器直接登录远程 Windows 服务器桌面环境。

**局限：**

1. 使用 RDP 客户端访问时需在终端上使用虚拟网 IP 地址和端口。

## 云网路由模式

将企业内物理局域网网段作为业务资源发布到 TM 安全云网，远程终端可直接从远端访问企业内网的局域网 IP 地址。

使用云网路由模式发布业务资源时：
1. 新建云网路由资源，选择 TMG 作为下一跳地址。
2. 设置将哪个局域网 IP 网段发布到 TM 安全云网，该网段作为路由策略下发到授权终端。
3. 设置 ACL 策略：网段中开放哪些 IP 地址、网络协议、端口号。
4. 将业务资源关联到云网路由资源（用于 TMA 显示），不关联则 TMA 不显示但终端仍可远程访问。

**优势：**

1. 远端用户无需更换访问目标 IP 地址，直接通过内网 IP 地址访问业务资源。
2. 可通过网段授权方式批量授权业务资源（如研发测试服务器集群），降低授权策略复杂度。
3. 终端用户在不改变任何终端配置的前提下，在任意地点使用原内网 IT 服务。

**局限：**

1. 终端用户在远程所处局域网的网络地址与远端数据中心或云 VPC 局域网地址不能有 IP 冲突。
2. TMG 缺少应用层访问控制能力（如 URI 限制、SSH 指令限制等）。

TMG 为云网路由资源提供两种转发模式：

- **SNAT 模式**（默认）：终端用户请求数据包到达 TMG 后，TMG 将源 IP 转为与业务资源连通的物理网 IP。企业网络环境不需要改造。局限是业务服务端收到的请求源 IP 为 TMG 的内网 IP。

- **路由转发模式**：TMG 不做任何处理直接转发数据包。业务服务端收到的源 IP 是每个终端的源 IP，可根据源 IP 区分访问来源（需在 TMG 上关闭源 IP 地址转换）。局限是需要改造内部网络，将该类资源应答数据包通过回指路由指向代理 TMG，且可能存在远端局域网 IP 地址与终端本地局域网 IP 地址冲突。
