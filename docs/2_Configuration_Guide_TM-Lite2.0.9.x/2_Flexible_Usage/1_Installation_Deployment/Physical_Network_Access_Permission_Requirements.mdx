TM 是部署在物理网络之上的虚拟化网络，各 TM 组件需要通过物理网络转发网络流量，因此您需要在部署前确认部署 TM 组件的服务器所分配物理IP地址能满足基本的网络访问权限。

## 快速部署配置

如果您希望快速部署 TM，建议您参考以下配置来开通物理网络访问权限。

### 外部网络访问权限

- 所有组件

  开放出方向的单向访问权限，确保每一个组件可以从内向外访问互联网。

- TMG组件额外配置
  
  在网络出口配置DNAT映射，将TMG的UDP9527端口映射到出口公网IP地址的任意一个固定的UDP端口，并允许任意地址可通过出口公网IP地址和映射端口正常访问TMG的UDP9527端口。

  用途：TMG将代理的所有业务资源和自身的管理接口全部收敛到UDP9527端口上，通过此端口统一转发所有业务访问流量，并对每一个网络4层数据包执行逐包认证和逐包加解密，因此需要TMG对外开放此端口的双向网络访问权限。

- TME组件额外配置（TMPro组件）

  在网络出口配置DNAT映射，将TME内网IP地址的TCP443端口以及计划发布的业务资源对应端口映射到出口公网IP地址的对应端口上，并允许任意地址通过出口公网IP地址的映射端口正常访问TME的对应端口。

  用途：终端用户在没有安装TMA的情况下，通过浏览器访问TME的Web Portal，并在认证授权后访问部署在企业内网的业务资源。
  
  注意：如果多个业务资源复用了同一个端口，则只需映射一次服务端口，如：多个业务资源都是使用TCP443端口提供HTTPS服务，则TME只需要对外映射一次TCP443端口即可。

### 内部网络访问权限

- 所有组件

  如所有组件部署在内部网络，则需要开通各个组件物理IP地址之间的访问控制策略，确保组件之间可以正常双向访问。

  开通管理员终端访问所有组件物理IP地址的TCP22端口权限，用于管理员访问各组件的控制台管理界面。

- TMC组件

  需开通管理员访问TMC物理IP地址的TCP16000端口权限；

  用途：管理员在内网通过TCP16000端口（HTTPS）访问TMC的Web管理页面。
  
- TMI组件（TMPro组件）
  
  需开通管理员访问TMI物理IP地址的TCP16004端口权限；
  
  用途：管理员在内网通过TCP16004端口（HTTPS）访问TMI的Web管理页面。


## 精确授权配置

出于安全考虑，TM 各组件需要精确的物理网络访问授权，建议您参考以下配置来开通物理网络访问权限。

### 外部网络访问权限

- 出方向网络控制策略

开放所有组件对 TM 云服务地址TCP8081端口与UDP9527端口的访问权限；

用途：TM 组件需要向 TM 云服务同步自身状态和反向连接方式。TM 为每一个企业客户提供云服务资源池（类似：xxx.TeloMatrixcloud.com），您可以在登陆 TM 企业注册平台后，在”我的云网“页面找到分配给您的云服务地址，如下图所示：

- 入方向访问控制策略
  
  - TMG组件

    在网络出口配置DNAT映射，将TMG的UDP9527端口映射到出口公网IP地址的任意一个固定的UDP端口，并允许任意地址可通过出口公网IP地址和映射端口正常访问TMG的UDP9527端口。
  
  - TME组件（TMPro组件）

    在网络出口配置DNAT映射，将TME内网IP地址的TCP443端口以及计划发布的业务资源对应端口映射到出口公网IP地址的对应端口上，并允许任意地址通过出口公网IP地址的映射端口正常访问TME的对应端口。
  
    用途：终端用户在没有安装TMA的情况下，通过浏览器访问TME的Web Portal，并在认证授权后访问部署在企业内网的业务资源。
  
    注意：如果多个业务资源复用了同一个端口，则只需映射一次服务端口，如：多个业务资源都是使用TCP443端口提供HTTPS服务，则TME只需要对外映射一次TCP443端口即可。

### 内部网络权限开通

- 所有组件

  开通管理员终端访问所有组件物理IP地址的TCP22端口权限，用于管理员访问各组件的控制台管理界面。

- TMC组件

  - 需开通管理员访问TMC物理IP地址的TCP16000端口权限。
  
    用途：管理员在内网通过TCP16000端口（HTTPS）访问TMC的Web管理页面。
  
  
  - 如果TMC与其他组件部署在同一个局域网内，则需要开通TMC与TMD、TMG、TME和TMI的双向UDP9527端口访问权限，TMC到TMI的TCP9092端口访问权限。

    用途：用于组件之间的联通数据通信。

- TMI组件（TMPro组件）
  
  - 需开通管理员访问TMI物理IP地址的TCP16004端口权限。
  
    用途：管理员在内网通过TCP16004端口（HTTPS）访问TMI的Web管理页面。
  
  - 组件联动网络权限
  
    需要开通TMI到TMC的TCP3306端口、TCP16000端口访问权限。

    用途：用于TMC与TMI之间的数据库同步和自动处置等联动业务。
