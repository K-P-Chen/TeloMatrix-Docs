---

TM 各组件部署在物理网络之上，需通过物理网络转发流量。部署前确认服务器物理 IP 地址满足以下网络访问权限。

## 快速部署配置

### 外部网络访问权限

- **所有组件**：开放出方向单向访问权限，确保各组件可从内向外访问互联网。

- **TMG 组件额外配置**：在网络出口配置 DNAT 映射，将 TMG 的 UDP9527 端口映射到出口公网 IP 地址的任意固定 UDP 端口，允许任意地址通过出口公网 IP 和映射端口访问 TMG 的 UDP9527 端口。

  用途：TMG 将所有代理业务资源和自身管理接口收敛到 UDP9527 端口，统一转发所有业务访问流量，对每个网络 4 层数据包执行逐包认证和逐包加解密。

- **TME 组件额外配置**（TMPro 组件）：在网络出口配置 DNAT 映射，将 TME 内网 IP 地址的 TCP443 端口及计划发布的业务资源对应端口映射到出口公网 IP 的对应端口，允许任意地址通过映射端口访问 TME 的对应端口。

  用途：终端用户未安装 TMA 时，通过浏览器访问 TME 的 Web Portal，认证授权后访问企业内网业务资源。

  注意：多个业务资源复用同一端口时只映射一次。如多个业务资源都使用 TCP443 端口提供 HTTPS 服务，TME 只需对外映射一次 TCP443 端口。

### 内部网络访问权限

- **所有组件**：开通各组件物理 IP 地址之间的访问控制策略，确保组件间正常双向访问。开通管理员终端访问所有组件物理 IP 地址的 TCP22 端口权限，用于管理员访问控制台管理界面。

- **TMC 组件**：开通管理员访问 TMC 物理 IP 地址的 TCP16000 端口权限。用途：管理员在内网通过 TCP16000 端口（HTTPS）访问 TMC 的 Web 管理页面。

- **TMI 组件**（TMPro 组件）：开通管理员访问 TMI 物理 IP 地址的 TCP16004 端口权限。用途：管理员在内网通过 TCP16004 端口（HTTPS）访问 TMI 的 Web 管理页面。

## 精确授权配置

出于安全考虑，各组件需精确的物理网络访问授权，建议参考以下配置。

### 外部网络访问权限

- **出方向网络控制策略**：开放所有组件对 TM 云服务地址 TCP8081 端口与 UDP9527 端口的访问权限。

  用途：TM 组件需向 TM 云服务同步自身状态和反向连接方式。TM 为每个企业客户提供云服务资源池（类似：xxx.TeloMatrixcloud.com），登录 TM 企业注册平台后在"我的云网"页面可查到分配的云服务地址。

- **入方向访问控制策略**：

  - **TMG 组件**：在网络出口配置 DNAT 映射，将 TMG 的 UDP9527 端口映射到出口公网 IP 地址的任意固定 UDP 端口，允许任意地址通过出口公网 IP 和映射端口访问 TMG 的 UDP9527 端口。

  - **TME 组件**（TMPro 组件）：在网络出口配置 DNAT 映射，将 TME 内网 IP 地址的 TCP443 端口及计划发布的业务资源对应端口映射到出口公网 IP 的对应端口，允许任意地址通过映射端口访问 TME 的对应端口。

    用途：终端用户未安装 TMA 时，通过浏览器访问 TME 的 Web Portal，认证授权后访问企业内网业务资源。

    注意：多个业务资源复用同一端口时只映射一次。

### 内部网络权限开通

- **所有组件**：开通管理员终端访问所有组件物理 IP 地址的 TCP22 端口权限，用于管理员访问控制台管理界面。

- **TMC 组件**：开通管理员访问 TMC 物理 IP 地址的 TCP16000 端口权限。用途：管理员在内网通过 TCP16000 端口（HTTPS）访问 TMC 的 Web 管理页面。

  如果 TMC 与其他组件部署在同一局域网内，开通 TMC 与 TMD、TMG、TME 和 TMI 的双向 UDP9527 端口访问权限，以及 TMC 到 TMI 的 TCP9092 端口访问权限。用途：组件间联通数据通信。

- **TMI 组件**（TMPro 组件）：开通管理员访问 TMI 物理 IP 地址的 TCP16004 端口权限。用途：管理员在内网通过 TCP16004 端口（HTTPS）访问 TMI 的 Web 管理页面。

  开通 TMI 到 TMC 的 TCP3306 端口、TCP16000 端口访问权限。用途：TMC 与 TMI 之间的数据库同步和自动处置等联动业务。
