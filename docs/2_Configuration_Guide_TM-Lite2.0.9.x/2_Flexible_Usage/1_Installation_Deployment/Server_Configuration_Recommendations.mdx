title: "服务器配置建议"
---

TM 各组件的 CPU、内存配置直接影响网络性能和系统稳定性。配置不足可能导致认证登录慢、策略生效时间长、访问速度慢等现象。根据计划部署规模，按照以下建议准备服务器。

## 最低配置

适用：产品测试或同时在线 500 人以下小规模部署

| 组件类型 | 服务器配置 | 性能指标 |
| ----- | ----- | ------ |
| TM 管理控制平台 TMC | CPU：4 核；内存：8GB；硬盘：500GB；网卡：千兆网卡 x 1 | 1. 支持 100 个用户同时认证；2. 支持管理最大 10000 个帐号；3. 支持连接管理最大 20 个 TM 网关 |
| TM 安全网关 TMG | CPU：4 核；内存：8GB；硬盘：500GB；网卡：千兆网卡 x 1 | 1. 支持 200 个业务资源；2. 支持最大 150000 并发会话数；3. 支持 300M 加密吞吐流量；4. 时延小于 0.04MS |
| TM 边缘网关 TME | CPU：8 核；内存：16GB；硬盘：500GB；网卡：千兆网卡 x 1 | 1. 支持 200 个业务资源；2. 支持最大 150000 并发会话数；3. 支持 300M 加密吞吐流量；4. 时延小于 0.04MS |
| TM 智能分析平台 TMI | CPU：16 核；内存：64GB；硬盘：24TB；网卡：千兆网卡 x 1 | 1. 日志处理性能 1.2 万条/秒；2. 日志存储时间约 3 个月（以 500 个终端，每终端每天产生 30 万条日志，每条日志 0.5KB 计算） |
| TM 安全网关 TMD | CPU：4 核；内存：8GB；硬盘：500GB；网卡：千兆网卡 x 1 | 1. 时延小于 10MS；2. 支持最大 1000 个解析地址；3. 支持 300M 加密吞吐流量 |
