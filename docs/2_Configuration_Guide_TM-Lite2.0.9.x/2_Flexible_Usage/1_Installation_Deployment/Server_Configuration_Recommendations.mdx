title: "服务器配置建议"
---

TM 各组件对CPU、内存在不同网络负载下有不同的要求，服务器硬件或虚拟机资源大小会直接影响到网络性能和系统稳定性。

如硬件配置不够则会在日常使用过程中出现认证登陆较慢、策略生效时间较长、网络访问速度慢等现象。

建议您根据计划的部署规模，按照以下服务器配置建议准备好服务器。

## 最低配置

适用：产品测试或同时在线500人以下小规模部署

|  组件类型  | 服务器配置 |  性能指标  |
| ----- | ----- | ------ |
| TM 管理控制平台 TMC |  CPU：4核；内存：8GB；硬盘：500GB；网卡：千兆网卡 x 1  | 1.支持100个用户同时认证;  2.支持管理最大10000个帐号;  3.支持连接管理最大20个 TM 网关 |
| TM 安全网关 TMG | CPU：4核；内存：8GB；硬盘：500GB；网卡：千兆网卡 x 1 | 1.支持200个业务资源；2.支持最大150000并发会话数；3.支持300M加密吞吐流量；4.时延小于0.04MS； |
| TM 边缘网关 TME | CPU：8核；内存：16GB；硬盘：500GB；网卡：千兆网卡 x 1 | 1.支持200个业务资源；2.支持最大150000并发会话数；3.支持300M加密吞吐流量；4.时延小于0.04MS； |
| TM 智能分析平台 TMI | CPU：16核；内存：64GB；硬盘：24TB；网卡：千兆网卡 x 1 | 1. 日志处理性能1.2万条/秒；2.日志存储时间大约3个月（以500个终端，每终端每天产生30万条日志，每条日志0.5KB计算）； |
| TM 安全网关 TMD | CPU：4核；内存：8GB；硬盘：500GB；网卡：千兆网卡 x 1 |1.时延小于10MS；2.支持最大1000个解析地址；3.支持300M加密吞吐流量。|
