## 功能说明

TM 安全云网中配置授权策略只需要指定什么人、用什么终端、访问什么业务即可，无需考虑物理位置或 IP 地址。

选择菜单【策略管理】->【授权策略】打开授权策略管理页面：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/1.png" style={{width: "100%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

点击"新增"按钮创建授权策略：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/2.png" style={{width: "90%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

## 第一步：配置基本信息

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/3.png" style={{width: "70%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

- **策略名称：**必填，不超过100个字符，只能由中英文、数字和英文点组合。
- **永久生效：**默认开启。关闭后需填写生效时间段。
- **备注：**非必填，最多200字符。

## 第二步：选择访问帐号

授权策略中需声明对哪些帐号和终端生效：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/4.png" style={{width: "80%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

- **筛选条件：**按帐号分类方式筛选帐号。分类方式包括主帐号组、帐号组、帐号角色、主帐号、帐号。详见 [TM 帐号分类](../2_Resource_Management/1_Account_Resource_Management/TM_Account_Classification)。
- **自动勾选子组：**勾选父组时同时勾选所有子组。若不勾选，则只勾选父组根目录下的帐号。
- **已筛选条件：**显示已选中的帐号，支持多种条件混合筛选。

## 第三步：选择使用终端

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/5.png" style={{width: "80%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

勾选方式与帐号一致。终端分类包括终端组、终端用途、终端类型、终端。终端用途含义参考 [终端用途](../2_Resource_Management/2_Endpoint_Resource_Management/Endpoint_Management#终端用途)。

## 第四步：选择目标业务

确定访问源后，选择可访问的业务资源：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/6.png" style={{width: "80%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

业务资源分类包括业务资源组、业务资源、业务资源服务。一个业务资源组包含多个业务资源，一个业务资源包含多个业务资源服务，一个业务资源可属于多个资源组。填写完帐号、终端与业务资源后，点击"保存"即可完成创建。

如需进一步限制应用层访问，可继续配置应用层控制策略。

## 第五步：选择应用层控制

当前版本支持 URI 黑名单和 SSH 指令黑名单两种应用层控制方式。

### URI 黑名单

点击 URI 黑名单开关开启过滤：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/7.png" style={{width: "80%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

资源列表显示第四步已选的 Web 应用资源，在操作列点击"选择URI集"为 Web 应用添加预定义的 URI 集合作为黑名单，一个资源可同时应用多个 URI 集。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/8.png" style={{width: "40%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

### SSH 指令黑名单

点击指令黑名单开关开启过滤：

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/9.png" style={{width: "80%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

在操作列点击"选择指令集"为 SSH 主机资源添加预定义的 SSH 指令集合作为黑名单，一个资源可同时应用多个指令集。

<img src="/images/2_Configuration_Guide_TM-Lite2.0.9.x/2_Flexible_Usage/3_Policy_Management/Authorization_Policy_Configuration/10.png" style={{width: "40%", display: "block", margin: "0 auto", border: "1px solid #e5e7eb", boxShadow: "0 2px 8px rgba(0,0,0,0.08)"}} />

URI 列表与指令集的编辑方法请参考[授权策略因子](Authorization_Policy_Factors)。
