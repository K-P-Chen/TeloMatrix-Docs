## 安装部署故障

### 现象：组件部署安装失败

**故障原因：**服务器配置及物理网络授权不满足条件。

**解决办法：**参考[服务器配置建议](../../2_Flexible_Usage/1_Installation_Deployment/Server_Configuration_Recommendations)和[物理网络访问权限要求](../../2_Flexible_Usage/1_Installation_Deployment/Physical_Network_Access_Permission_Requirements)调整服务器配置和网络访问权限。

### 现象：TM 组件安装部署后，同步证书失败

**故障原因：**

- 原因 1：TM 组件无法正常访问互联网。
- 原因 2：TM 组件数量超出授权范围。
- 原因 3：TMC 正常部署，其他组件同步证书失败，可能是 IP 地址冲突。

**解决办法：**

- 办法 1：在 TM 组件控制台使用"网络诊断"检查是否能访问互联网，不能则检查网络配置和访问授权。
- 办法 2：在 TMC【状态监控】->【云网监控】页面注销不再使用的组件后重试。如需要更多授权请升级为 TMPro 版本。
- 办法 3：在 TM 组件控制台管理界面检查分配的虚拟管理 IP 是否与本地 IP 冲突，如有冲突请联系修改。

## 业务访问故障

### 现象：终端用户访问业务资源正常，但访问互联网很慢甚至失败

**故障原因：**终端使用内网 DNS 或 TMD 进行域名解析时可能出现。互联网网站采用 CDN 加速且各运营商有区域优化，不同地区终端解析同一域名结果不同。使用内网 DNS 或 TMD 解析导致所有终端得到相同解析结果，访问慢或失败。

**解决办法：**业务资源使用域名代理方式发布，在认证模板中设置域名解析为 HOST 模式。终端使用本地 DNS 解析互联网域名，企业内网业务通过本地 HOST 文件解析。此方法对 Windows 和 Mac 版 TMA 有效，对移动端 TMA 无效。

### 现象：AD/LDAP 认证登录失败

**故障原因：**AD/LDAP 帐号使用大写（AD/LDAP 对帐号大小写不敏感），TMA 上使用小写帐号登录。TMC 对帐号大小写敏感，大小写帐号被识别为两个不同帐号。

**解决办法：**在 TMC AD/LDAP 同步设置中，将"帐号存储字符转换"选项设置为"转小写"。同步时所有帐号转为小写存储，TMA 用户可使用小写帐号登录。

### 现象：登录客户端后访问部分业务资源失败

**故障原因：**

- 原因 1：TMG 状态异常。
- 原因 2：TMG 与业务资源网络通信异常。

**解决办法：**

- 办法 1：登录 TMC，在【状态监控】->【云网监控】页面检查 TMG 状态，排查异常原因，确认物理网络访问授权正常、TMG 服务正常。
- 办法 2：在【状态监控】->【云网监控】页面打开 TMG 远程管理界面，在"网络管理"标签页使用"网络测试"工具检查 TMG 与代理业务资源的访问状态。如无法访问，在物理网络上排查 TMG 到代理业务资源的网络访问权限。

### 现象：客户端访问云网路由模式下的业务资源失败

**故障原因：**终端用户本地网络 IP 地址与业务资源所在物理网络 IP 地址段冲突。

**解决办法：**确认是否存在 IP 冲突。如存在：

- a. 将业务资源改为代理模式发布到 TM 安全云网。
- b. 通知终端用户调整本地网络 IP 地址段，或调整业务资源所在网络 IP 地址段，使两者不再冲突。
