## 安装部署故障

### 现象：组件部署安装失败

**- 故障原因：**

  服务器配置以及服务器的物理网络授权不满足条件

**- 解决办法：**

  请参考[服务器配置建议](../../Flexible_Usage/Installation_Deployment/Server_Configuration_Recommendations)和[物理网络访问权限要求](../../Flexible_Usage/Installation_Deployment/Physical_Network_Access_Permission_Requirements)调整服务器配置和网络访问权限。


### 现象：TM 组件安装部署后，同步证书失败

**- 故障原因：**
  
  原因1：TM 组件无法正常访问互联网。

  原因2：TM 组件数量已经超出授权范围。

  原因3：如果TMC能正常部署，其它组件同步证书失败，则可能是IP地址冲突导致。

**- 解决办法：**

  办法1：在 TM 组件的控制台使用“网络诊断”功能检查组件网络是否能够访问互联网，如不能访问则需要检查 TM 组件的网络配置和网络访问授权是否正常。

  办法2：在TMC上【状态监控】->【云网监控】页面注销不再使用的组件，然后重试。如果确实需要更多 TM 组件授权，请联系我们将您的授权类型升级为 TMPro版本。

  办法3：在 TM 组件的控制台管理界面检查分配的虚拟管理IP是否与本地IP地址冲突，如有冲突，请联系我们进行修改。

## 业务访问故障

### 现象：终端用户访问业务资源正常，但访问互联网很慢甚至失败

**- 故障原因：**

  当终端使用内网DNS或使用TMD进行域名解析时可能出现此现象。

  互联网上的网站为了优化访问效率，通常会采用CDN进行加速，各运营商也会对各自服务区域内的网站进行优化或者限制，因此当位于不同地区的终端在访问同一个互联网网站时域名解析的结果会不相同。

  在 TM 安全云网中，终端可能位于全球任意位置，使用内网DNS或TMD进行域名解析使得所有终端得到的解析结果都一样，导致网站访问慢甚至无法访问。
  
**- 解决办法：**

  业务资源使用域名代理方式发布，同时在认证模板中设置域名解析为HOST模式。这时终端将使用本地获取的DNS解析互联网访问域名，企业内部业务资源访问使用本地HOST文件进行解析。

  这种方法对Widows和Mac版本的TMA有效，对移动端TMA无效。

### 现象：AD/LDAP认证登录失败

**- 故障原因：**

  AD/LDAP的帐号使用了大写（AD/LDAP对帐号大小写不敏感），而在TMA上使用小写帐号进行登录。

  TMC对帐号大小写敏感，小写帐号与大写帐号在TMC上将被识别为两个不同帐号。

**- 解决办法：**

  在TMC的AD/LDAP的同步设置里，将“帐号存储字符转换”选项设置为“转小写”，即当TMC同步AD/LDAP帐号时将所有帐号都转为小写存储，认证时TMA用户可以使用小写帐号进行认证登录。

### 现象：登录客户端后访问部分业务资源失败

**- 故障原因：**

  原因1：TMG状态异常。

  原因2：TMG与业务资源网络通信异常。
  
**- 解决办法：**

  办法1：登录TMC，在【状态监控】->【云网监控】页面，检查TMG的状态，如有异常需要排查TMG异常原因，确认TMG的物理网络访问授权正常，TMG服务正常。

  办法2：登录TMC，在【状态监控】->【云网监控】页面打开TMG的远程管理界面，在网络管理标签页使用“网络测试”工具检查TMG与代理业务资源的访问状态。如无法访问则需要在物理网络上排查TMG到代理业务资源的网络访问权限是否正常。

### 现象：客户端访问云网路由模式下的业务资源失败

**- 故障原因：**

  终端用户的本地网络IP地址与业务资源所在的物理网络IP地址段冲突。

**- 解决办法：**

  请确认终端用户的本地网络与业务资源所在网络是否存在IP地址冲突问题。

  如果存在可以通过以下两种方式解决：

  a. 将业务资源改为使用代理模式发布到 TM 安全云网。

  b. 通知终端用户调整本地网络IP地址段，或调整业务资源所在网络的IP地址段，使两者不再冲突。
